# 轻游台维护说明

## 日常添加小游戏

1. 把新的单文件游戏放到 `outputs/` 目录，例如 `my-game.html`。
2. 在 `outputs/data/games.json` 里新增一条记录。
3. 打开大厅页，点右上角刷新按钮确认游戏出现。

## 清单字段

- `id`: 稳定唯一标识，只用小写字母、数字和短横线。
- `title`: 游戏名。
- `subtitle`: 一句话玩法类型。
- `description`: 游戏卡片说明。
- `url`: 游戏文件相对路径，不要写外部网址。
- `status`: 推荐使用 `可玩`、`测试中`、`维护中`。
- `version`: 游戏版本号。
- `updated`: 更新时间，格式 `YYYY-MM-DD`。
- `tags`: 搜索和分类标签。
- `controls`: 操作说明，后续可用于详情页。
- `accent`: 卡片主色，使用十六进制颜色。

## 安全约定

- 首页不使用外部 CDN、广告脚本或统计脚本。
- `index.html` 设置了基础 CSP，只允许加载本目录资源。
- 游戏入口会拒绝 `http://`、`https://` 和 `//` 这类外部链接。
- 新游戏如果需要联网，建议先单独审查，再加入大厅。

## 部署建议

- 局域网试玩：在 `outputs/` 目录启动静态服务，绑定 `0.0.0.0`。
- Windows 本地启动：右键运行 `start-arcade.ps1`，或在 PowerShell 里执行它。
- 大陆长期访问：优先放到国内云、国内 CDN 和已备案域名。
- 公网商业化：完成 ICP 备案、公安联网备案、HTTPS、隐私政策、用户协议。
- 海外试玩：可以使用 GitHub Pages、Cloudflare Pages、Netlify、Vercel，但不建议作为大陆商业主站。
- 版本维护：每次调整游戏节奏或规则时更新 `version` 和 `updated`。

## 商业化维护

- 广告接入前，先设计广告配置和频控，不要直接把第三方脚本硬写进游戏。
- 充值接入前，先建设后端订单、支付回调、实名、防沉迷和对账。
- 上架平台前，单独维护平台适配层，避免游戏核心逻辑被平台 SDK 绑死。
- 每次上线记录到 `PROJECT_PROGRESS.md`，包括版本、改动、风险和下一步。
